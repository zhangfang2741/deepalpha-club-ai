# 机构资金信号 · 判断逻辑说明书

> 对应代码：`app/services/institutional_signals/`（constants / fetchers / dimensions / states / calculator）
> 目的：把「数据 → 信号 → 维度分 → 状态」的每一步阈值与公式讲清楚，可复核、可调参。

---

## 0. 总览：一条决策链，五个问题

| 维度 | 核心问题 | 状态 | 数据源 |
|------|----------|------|--------|
| Expectation 预期 | 分析师在重新定价未来吗？ | ✅ 已接入 | FMP |
| Positioning 仓位 | 资金已经在下注了吗？ | ✅ 已接入（快照） | yfinance 期权 |
| Participation 参与度 | 其他资金跟了吗？ | ✅ 已接入 | FMP 日线 |
| Fundamental 基本面 | 经营真的变好了吗？ | ✅ 已接入 | FMP earnings |
| Confirmation 确认 | 长期资金确认了吗？ | ✅ 已接入 | FMP insider |

每个维度输出 **0–100 子分**；缺失维度按**中性 50 计入**综合分（不剔除），并计入**覆盖度**与**置信度**。

**维度状态语义**（决定是否计入覆盖度）：
- `ok`：有真实数据，正常打分。
- `partial`：有部分数据但不完整（如只有未来财报日、无历史 EPS）。计入覆盖度。
- `unavailable`：**无可用数据**（拉取失败或该源为空）。**不计入覆盖度**——避免"零数据却显示高置信度"。

---

## 1. Expectation 预期

**数据源**：FMP `price-target-summary`（目标价近月/季/年均值 + 家数）、`grades-historical`（各月强买/买/持有/卖出家数）。

**基准分 50**，逐项加减：

| 信号 | 公式 | 阈值 → 加减分 | hit |
|------|------|---------------|-----|
| 目标价修正 | `mom = (近月均值 − 近季均值) / 近季均值` | ≥ +5% → +20；≥ +1% → +12；≤ −1% → −15 | ↑上调时 |
| 评级共识 | `买入占比 = (强买+买入) / 总家数`，比最近两个有效月 | 环比 +2pt → +15；环比 −2pt → −15；占比 ≥ 60% → +8 | 环比上升时 |
| EPS/营收修正 | — | 待每日快照库（Phase 4） | 恒 false |

**门槛**：目标价样本数 `lastMonthCount < 2` 不采信。两项均无有效数据 → `status=partial`。

## 2. Positioning 仓位（期权快照）

**数据源**：yfinance `option_chain`，聚合 **45 天内近月合约**（不足 2 个到期日则取最近 2 个）。

> ⚠️ 关键限制：yfinance 是**快照**，只有当下的量/OI/IV。手册里的「OI↑ / IV↑ / IV Rank」是**变化率**，需历史基线。本版用快照能算实的三个量，变化率与 Rank 标注为「待每日快照库」。

聚合量：`call_vol / put_vol / call_oi / put_oi`（求和），`atm_iv`（行权价最接近现价的 Call+Put 的 IV 均值，年化小数）。

**基准分 50**：

| 信号 | 公式 | 阈值 → 加减分 | hit |
|------|------|---------------|-----|
| Call 资金流 | `PCR量 = put_vol/call_vol`；`量/仓 = call_vol/call_oi` | PCR量 ≤ 0.7 **且** 量/仓 ≥ 0.4 → +20；PCR量 ≤ 0.9 → +8 | 强看涨下注时 |
| 看跌压力 | `PCR量` 与 `PCR仓 = put_oi/call_oi` | 任一 ≥ 1.3 → −15 | put 主导时（方向↓） |
| 隐含波动率 | `atm_iv` | ≥ 60% → +6；≥ 40% → +3 | IV 偏高时 |

**读法**：`量/仓比` 高 = 当日新增下注活跃（「有资金提前下注」的快照代理）；PCR 低 = 看涨；IV 高 = 市场预期有事件。

## 3. Participation 参与度

**数据源**：FMP `historical-price-eod`（近 60 天日线）。窗口 = 不含当日的**前 20 个交易日**。

**基准分 50**：

| 信号 | 公式 | 阈值 → 加减分 | hit |
|------|------|---------------|-----|
| 相对成交量 | `当日量 / 20日均量` | ≥ 2.0 → +25；≥ 1.5 → +15；≥ 1.2 → +8；< 0.8 → −10 | ≥ 1.5 时 |
| 跳空幅度 | `(当日开 − 昨收)/昨收` | ≥ +2% → +10；≤ −2% → −10 | 幅度达标时 |
| 价格突破 | 当日收盘 vs 20日高/低 | 破 20 日高 → +15；破 20 日低 → −15 | 突破/跌破时 |

历史不足 21 天 → `status=partial`。

## 3b. Fundamental 基本面

**数据源**：FMP `earnings-calendar`（历史 epsActual/epsEstimated/revenueActual/revenueEstimated + 未来财报日）。

**基准分 50**：

| 信号 | 公式 | 阈值 → 加减分 | hit |
|------|------|---------------|-----|
| 财报兑现 | 最近若干季 `epsActual > epsEstimated` 的**连续** Beat 数 | 连 ≥3 → +22；连 2 → +14；连 1 → +6；最近一季 Miss → −12 | 连 ≥2 或 Miss 时 |
| 营收超预期 | `(revenueActual − revenueEstimated)/revenueEstimated`（最近季） | ≥ +2% → +12；≤ −2% → −12 | 幅度达标时 |
| 财报窗口 | 距下次财报天数 | 0–21 天 → +5 | 在窗口内时 |
| 指引方向 | — | 需 Transcript NLP（后续） | 恒 false |

> Guidance「比 Beat 更重要」，但方向判定需对 Transcript 做 NLP，暂缺；用「连续兑现」承接基本面强度。

## 3c. Confirmation 确认

**数据源**：FMP `insider-trading/statistics`（季度：`totalPurchases` 开放市场买入、`acquiredDisposedRatio` 增/减比、`totalSales`）。聚合最近 **2 季**。

**基准分 50**：

| 信号 | 公式 | 阈值 → 加减分 | hit |
|------|------|---------------|-----|
| 内部人交易 | 优先看 `totalPurchases`（真买入）；其次 `acquiredDisposedRatio` | 有开放市场买入 → +20；增/减比 ≥ 1.2 → +8；增/减比 ≤ 0.3 且卖出 ≥ 10 笔 → −12 | 买入 / 集中减持时 |
| 13F 机构持仓 | — | 需 FMP Ultimate 版（后续） | 恒 false |
| ETF 资金流 | — | 行业级，复用 ETF 模块（后续） | 恒 false |

> 手册「Insider Sell 参考意义低于买入」——故买入强加分，减持仅在**集中**时才小幅扣分，避免高管常规卖出误报。

## 4. 综合分与置信度（本次修正的核心）

**综合分**（权重）：预期 0.30 · 仓位 0.25 · 参与度 0.20 · 基本面 0.15 · 确认 0.10。

```
composite = Σ(维度分 × 权重) / Σ权重
```

**修正点**：缺失维度**不再剔除**，按中性 50 计入 → 半盘信息不会拉出满分结论。
例：预期 82、参与度 73，其余三维缺失（各 50）→ 综合分 ≈ 64（而非旧算法的 78）。

**覆盖度** = status 为 ok/partial 的维度数（满分 5）。
**置信度**：覆盖 ≥ 4 → 高；≥ 2 → 中；否则低。

## 5. 状态引擎（组合 → 标签）

状态由各维度的 **hit 布尔量 + 方向**组合触发，附证据链。命中多个则全列；无命中 → ⚪ 中性。

| 状态 | 触发条件（代码口径） | 星级 |
|------|----------------------|------|
| 📈 预期上修 | 目标价↑hit **且** 评级共识↑hit | ★★★★★ |
| 🚀 趋势确认 | 相对成交量↑hit **且** 价格突破↑hit **且** 预期分 ≥ 55 | ★★★★ |
| 🔥 机构建仓 | 预期分 ≥ 55 **且** Call资金流↑hit **且** IV↑hit（放量为可选加分，非门槛） | ★★★★★ |
| 💰 聪明钱 | 预期分 ≥ 55 **且** Call资金流↑hit **且** IV↑hit **且** 未突破 | ★★★★★ |
| ⚡ 热钱 | Call资金流↑hit **且** IV↑hit **且** 预期分 < 55 | ★★★★ |
| 🌱 基本面改善 | 营收超预期↑hit **且** 财报兑现↑hit（目标价↑为可选确认，非门槛） | ★★★★★ |
| ❄ 机构派发 | 评级共识↓hit **且**（看跌压力↓hit 或 内部减持↓hit） | ★★★★ |

**设计原则：「提前/早」的信号不该要求「市场已确认」的证据。** 放量、目标价上调都是市场**反应后**才出现的，把它们设成硬门槛会排除掉最有价值的「提前」信号。故：
- 🔥机构建仓：核心=预期+期权看涨+IV，放量为可选加分（「悄悄吸筹」本就不放量）。
- 💰聪明钱：机构建仓 + 价格尚未突破（即「建仓且你还很早」，赔率最好）；**须有预期背书**，否则与 ⚡热钱（投机）无法区分。
- 🌱基本面改善：核心=营收超预期+连续兑现，目标价上调为可选确认（分析师未上调恰是「早」）。

## 5b. 机构建仓榜（发现型入口）

**两段式**：全 universe 逐支跑完整五维不现实（期权抓取太慢），故：
- **筛选（榜单）**：扫描 universe，每支只用 **4 个 FMP 维度**（跳过期权），按综合分排名。
- **确认（详情）**：用户点进某支，跑**完整五维**（含 yfinance 期权）。

**universe（动态）**：取自 FMP `sp500-constituent`（S&P 500 全量成分股，过滤含点号多类股）；拉取失败降级到 `SCAN_UNIVERSE_FALLBACK`（~40 精选大盘股）。**不再写死**。

**扫描如何不阻塞请求**：stale-while-revalidate + 后台任务
- 命中缓存 → 立即返回；
- 缓存过期（> 6h）→ 先返回旧数据，同时后台 `asyncio` 任务重扫（带 Redis 锁防并发）；
- 无缓存 → 后台开扫并返回 `status=computing`，前端每 15s 轮询直到 ready。
- 因扫描在后台，故可覆盖**全量 S&P 500**（并发 20），而非受单次请求耗时限制。

**排序口径**：`(是否有偏多状态, 综合分)` 降序。综合分与详情页一致（缺失维度按中性计入）——仓位对所有被扫股票同权重缺失，不影响相对排序。只有 `institution_accumulation / expectation_upgrade / breakout_confirmation / fundamental_turn / smart_money` 计为"上榜偏多状态"（撤退等看空状态不进建仓榜）。

`GET /api/v1/institutional-signals/leaderboard`。

> ⚠️ 后台任务跑在单 web 进程内——全量扫描会占用该进程数分钟。生产级做法是迁到独立 worker / 定时任务（Celery beat 或外部 cron 命中该端点预热），列为运维项。

## 5c. 每日快照库（把「水平」升级为「变化率」）

`signal_snapshots` 表：每支标的每天一行（`symbol + snapshot_date` 唯一），记录 EPS/营收一致预期、目标价均值、期权 OI/量/ATM IV。详情页每次计算（缓存未命中）时 **best-effort 写入当日快照**（DB 不可用不影响报告）。

据历史快照衍生的**变化率信号**（`deltas.py` 纯函数）：
| 信号 | 公式 | 触发 |
|------|------|------|
| **IV Rank** | current 在近一年 IV 的 `(cur−min)/(max−min)×100`；历史点 < 20 → None | Rank ≥ 70 → hit（替代原「水平」启发式） |
| **Call OI 变化** | current vs 近 5 日前 OI 的 `pct_change` | ≥ +10% → hit up（真·新增仓位） |
| **EPS/营收修正** | current 一致预期 vs 近 90 天前 | ≥ +1% → hit up |

历史不足时各自返回 None，维度**回退到水平口径**（向后兼容，不报错）。快照随天数累积后自动生效。
